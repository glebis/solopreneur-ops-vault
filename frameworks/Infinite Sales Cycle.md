# Infinite Sales Cycle

> Sales never stops. Every output feeds the next input.

## The Model

Most solopreneurs treat sales as a campaign: launch → push → rest → repeat. This creates boom-bust cycles and morning confusion ("what should I sell today?").

The infinite cycle treats every action as both a delivery AND a lead generator:

```
Create → Publish → Engage → Convert → Deliver → Document → Create
  ↑                                                           |
  └───────────────────────────────────────────────────────────┘
```

## Each Step Feeds the Next

| Step | Action | Sales Output |
|------|--------|-------------|
| **Create** | Build a skill, write a post | Portfolio piece |
| **Publish** | Share on LinkedIn/YouTube/site | Visibility |
| **Engage** | Reply to comments, DMs | Relationship |
| **Convert** | Offer help, invite to lab | Lead → Sale |
| **Deliver** | Run the lab/community | Case study material |
| **Document** | Record results, testimonials | Social proof |

## Daily Activation

The [[Daily Dashboard]] always includes items from at least 2 different cycle stages. This prevents the common solopreneur trap of spending all time in "Create" and none in "Engage" or "Convert."

**Anti-pattern:** Building 50 skills (Create) without promoting them (Publish/Engage).

## Agent Integration

Agents handle the parts of the cycle that don't require personal voice:
- **Research** for Create (market scans, competitor analysis)
- **Scheduling** for Publish (optimal timing)
- **Monitoring** for Engage (new comments, mentions)
- **Briefing** for Convert (lead context before calls)
- **Summarizing** for Document (session highlights)

## Revenue Targets (Berlin 2026)

For a solo educator in Berlin, baseline sustainability:

| Item | Monthly |
|------|---------|
| Living costs Berlin | ~2,500 EUR |
| Tools & subscriptions | ~300 EUR |
| Tax reserve (30%) | ~1,200 EUR |
| **Minimum revenue** | **~4,000 EUR** |
| **Comfortable target** | **~7,000 EUR** |

This means: 1 cohort (6-8 people × 500-800 EUR) per quarter + community revenue.
