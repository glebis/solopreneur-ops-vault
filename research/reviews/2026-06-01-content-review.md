## Content Trends

### LinkedIn Algorithm — Comment Weight Now Confirmed: 15× Likes

The May 15 review documented saves driving 5× more reach than likes. New data this period:

**Comments now carry 15× the algorithmic weight of likes.** This is not metaphorical — it reflects how LinkedIn 360 Brew AI scores engagement signals: a comment signals "I read this, formed a thought, and wrote it out," which is a much higher-cost action than a thumbs-up. Practical implication: a post with 10 meaningful comments outperforms a post with 150 likes on reach and distribution.

**Current gap in vault:** Neither the Hook Patterns nor the Calendar framework mentions comments as the primary metric to optimise for. CTAs currently say "engage" generically. Updated below.

### LinkedIn Algorithm — Interest Graph Has Replaced Social Graph

The shift from Social Graph (content from people you know) to Interest Graph (content based on what you engage with, regardless of connection) is now confirmed. This has two implications not previously documented:

1. **Follower count is a lagging indicator.** A creator with 500 highly engaged followers in the right niche can outreach a creator with 5,000 mixed-network followers, because the Interest Graph routes to interested non-followers.

2. **Your profile validates your content.** The algorithm cross-references every post with your headline, About section, and work history. A post about "AI for non-developers" from someone whose profile says "AI educator | Claude Code | 350+ alumni" gets a distribution multiplier. The same post from someone whose profile says "entrepreneur | builder | curious human" gets no validation boost. Profile alignment is now a distribution lever, not just a branding choice.

**Current gap in vault:** No framework addresses profile alignment as an algorithmic lever. The Interest Graph framework likely covers some of this — but the connection between profile text and per-post distribution is not in the active frameworks. New concept below.

### LinkedIn Algorithm — Hashtags Officially Demoted

LinkedIn has demoted hashtags from a reach driver to a "lightweight topic signal." The updated guidance: 0–3 hashtags per post. More than 3 adds no benefit and may signal low-quality content.

**Current vault gap:** No framework mentions hashtags. Many posts in the wild (including from this account's recommended patterns) use 5–10 hashtags. Updated in Hook Patterns below.

### AI Education — The Judgment Gap in 2026 Cohorts

In 2025 AI cohorts, live session time was used to explain foundations: what agents are, why context windows matter, how prompts work. In mid-2026, cohort participants arrive having watched the videos, read the posts, and tried the tools. **The conceptual foundation is mostly pre-loaded.**

What 2026 participants lack is **judgment**: when to use which approach, how to evaluate whether agent output is actually good, what to do when the workflow breaks in production. Live cohort time is now most valuable when it concentrates on decision-making under real conditions — with the educator making choices and narrating reasoning — rather than explanation.

**Implication for this operation:** The Claude Code Lab's live session structure should shift the framing from "here's how this works" to "here's how I think through this, watch the decision." Pre-work carries the conceptual load; live time develops judgment.

### Solopreneur Stack — Agent Routing Is Now Standard Practice

The emerging best practice for solopreneurs in 2026 is not "use one AI tool" but **route tasks to the right model by effort level:**

| Task Type | Recommended Model | Why |
|-----------|------------------|-----|
| Deep reasoning, architecture decisions | Claude Opus 4.8 | Max reasoning quality |
| Content drafting, analysis, frameworks | Claude Sonnet 4.6 | Best balance of quality/cost |
| High-volume, structured, repeatable | Claude Haiku 4.5 | Fast, cheap, accurate enough |

The AI budget for a working solopreneur running this stack is ~$50–200/month. The multi-agent system — Perplexity for research, Claude Sonnet for drafting, automation layer for scheduling — is now documented as a content creation standard, not an advanced pattern.

**Vault gap:** `frameworks/Content Leverage.md` describes what agents do but not which models to use for which tasks. Updated below.

## New Concepts

### 1. Profile Authority Alignment

**The idea:** LinkedIn's 360 Brew AI doesn't just score post quality — it cross-references post content with your profile (headline, About section, work history) to validate topical authority. A post that matches your declared expertise gets a distribution multiplier. A post that doesn't match — even if it's excellent — gets routed more narrowly.

**Framework core:**
- Your profile is the authority certificate that LinkedIn checks for every post you publish
- Headline and About section should use the exact keywords of your topic DNA: "AI tools," "non-developer," "Claude Code," "MCP," "cohort"
- Work history descriptions should reflect demonstrated expertise, not just roles
- Audit cadence: check profile alignment whenever you publish a new post type or pivot to a new topic area

**Saveable structure for a post:** "Your profile is the only thing on LinkedIn that works for you while you sleep. Here's how to align it with the 360 Brew AI interest graph in 4 elements: [headline / About / featured / skills]"

**Candidate file:** `frameworks/Profile Authority Alignment.md`

### 2. Judgment-First Cohort Design

**The idea:** In 2026, AI cohort participants arrive with pre-loaded conceptual knowledge. Live session value is no longer in explanation — it's in judgment development: watching an expert make real decisions in real time, with narrated reasoning. This requires a redesign of how live session time is structured.

**Framework core:**
- Pre-work carries conceptual load (videos, readings, tool setup)
- Live session = "expert practitioner making real decisions under observation"
- Format shift: less lecture, more narrated live problem-solving ("I'm going to build X from scratch right now, here's how I think about each decision")
- Participant role: active decision-making, not passive watching (what would you do here?)
- Measurement: participant judgment quality (can they debug their own work?) > concept test scores

**Implications for Claude Code Lab:**
- Replace tutorial walkthroughs with live production sessions where the educator builds real things
- Frame cohort sessions as "judgment development labs," not "skill lessons"
- Outcome statement: "You'll leave able to evaluate AI output, debug workflows, and decide when to use which approach" — not "you'll learn prompt engineering"

**Candidate file:** `frameworks/Judgment-First Cohort Design.md`
