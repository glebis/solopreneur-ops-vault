# Conference Talk Outline (20 min)

## Title

"[Contrarian hook] — [Specific result]"

Example: "Stop Learning Prompt Engineering — Build AI Agents Instead"

## Structure

### 0:00-2:00 — Hook
Contrarian statement or surprising result that challenges the audience's assumptions.

"Most AI training teaches you to talk to AI. I teach people to build AI systems that work while they sleep. Here's a live demo."

### 2:00-12:00 — Live Demo
Build something real in 10 minutes. The audience watches a working system emerge.

**Good demos:**
- Vault → research → site in 10 min
- Voice dictation → agent → published content
- LinkedIn scan → engagement tasks → dashboard

**Rules:** No slides during demo. Screen share + narration only.

### 12:00-17:00 — Framework
The mental model behind what they just saw. One framework, one diagram.

Examples:
- [[frameworks/Inner Outer Loop]] — what you do vs what agents do
- [[frameworks/Behavioral Activation]] — action before motivation
- [[frameworks/5-Level Autonomy]] — from copilot to observer

### 17:00-19:00 — Key Takeaway
One sentence they'll remember tomorrow. Repeat it twice.

### 19:00-20:00 — CTA
QR code on screen → free resource (Context Interview template, skills catalog, or community invite).

"Scan this → get the template I just used → try it yourself tonight."

## Capture Optimization

From [[frameworks/Conference Speaking Pipeline]]: 56% capture rate benchmark.
- QR code visible for full 60 seconds
- Resource must be immediately useful (not "sign up for newsletter")
- Follow up within 24 hours on LinkedIn

## Checklist

- [ ] Talk title passes the "would I click this at a conference?" test
- [ ] Demo works offline (conference WiFi is unreliable)
- [ ] QR code tested on phone
- [ ] Backup slides ready if demo fails
- [ ] Follow-up sequence prepared (LinkedIn message template)
